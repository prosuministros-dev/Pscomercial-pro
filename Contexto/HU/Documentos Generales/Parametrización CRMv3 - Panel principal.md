| 2 | Panel de pedidos |  |  |  |
|  | 2.1 | Este panel debe mostrar: el estado del pedido, numero de pedido, cantidad pendiente por comprar, cliente, fecha y hora, responsable, asunto, subtotal y moneda negociación. Esta información ningun usuario lo podra modificar o editar. |  |  |
|  |  | 2,1,1 | Estado de pedido: |  |
|  |  |  | Un pedido puede tener 4 estados: En proceso, cerrado, perdido y anulado. En el panel principal unicamente se deben visualizar los pedidos que estan en proceso. |  |
|  |  | 2,1,2 | Numero de pedido: |  |
|  |  |  | Cuando el comercial pasa la cotización a pedido, el sistema debe asignar un consecutivo de manera automatica (comienza apartir del numero 20.000). |  |
|  |  |  | Los pedidos que se visualicen en el panel principal deben estar organizados por el numero de pedido en orden descendente |  |
|  |  | 2,1,3 | Cantidad pendiente por comprar |  |
|  |  |  | En esta columna se debe visualizar cuantos productos estan pendientes por comprar. Tener en cuenta que si un producto pendiente por comprar tiene 5 unidades, estas 5 unidades deben aparecer en esta columna. Por otro lado, si tenemos dos productos pendientes por comprar y cada producto tiene 3 unidades, estas 6 unidades deben aparecer en esta columna. |  |
|  |  | 2,1,4 | Cliente |  |
|  |  |  | Razon social del cliente |  |
|  |  | 2,1,5 | Fecha y hora |  |

|  |  | 2,1,6 | Responsable |  |

|  |  | 2,1,7 | Asunto |  |
